# 공통 리스트 화면 템플릿

> 이 템플릿은 Compli-Law 서비스의 **모든 리스트 화면에 공통 적용**되는 레이아웃·컴포넌트 구조입니다.
> 신규 리스트 화면 작성 시 이 파일을 복사한 후 `[대괄호]` 항목만 채워 사용하세요.

---

## 전체 레이아웃 구조

```
┌─────────────────────────────────────────────────────────────────────────┐
│ ZONE-1: 최상단 바                                                        │
├─────────────────────────────────────────────────────────────────────────┤
│ ZONE-2: 섹션 헤더 (제목 + 기능버튼)                                     │
├─────────────────────────────────────────────────────────────────────────┤
│ ZONE-3: 필터 Area                                                        │
├─────────────────────────────────────────────────────────────────────────┤
│ ZONE-4: 테이블 상단 (결과 수 + 테이블 도구)                              │
├─────────────────────────────────────────────────────────────────────────┤
│ ZONE-5: 데이터 테이블                                                    │
├─────────────────────────────────────────────────────────────────────────┤
│ ZONE-6: 페이지네이션                                                     │
└─────────────────────────────────────────────────────────────────────────┘
```

---

## ZONE-1: 최상단 바

```
┌─────────────────────────────────────────────────────────────────────────┐
│  [페이지 제목]                   (?)(사용자명) (소속)                    │
│                         [상위메뉴] > [현재메뉴]                          │
└─────────────────────────────────────────────────────────────────────────┘
```

| 요소 | 위치 | 설명 |
|------|------|------|
| 페이지 제목 | 좌측 | 현재 화면의 주 기능명 (예: "BOM 마스터") |
| 브레드크럼 | 우측 상단 | 상위 메뉴 > 현재 메뉴 계층 표시 |
| 도움말 아이콘 | 우측 | `?` 버튼 — 화면별 도움말 팝업 |
| 사용자 정보 | 우측 끝 | 로그인 사용자명 + 소속 |

---

## ZONE-2: 섹션 헤더

```
┌─────────────────────────────────────────────────────────────────────────┐
│  [섹션 제목]                                          [기능버튼 영역]    │
└─────────────────────────────────────────────────────────────────────────┘
```

| 요소 | 위치 | 규격 |
|------|------|------|
| 섹션 제목 | 좌측 | `font-size: 1.125rem`, `font-weight: 600` |
| 기능버튼 영역 | 우측 | 버튼 그룹 (Primary 버튼 최대 1개, Secondary는 복수 허용) |

### 기능버튼 우선순위 배치 (우→좌 순서)

| 우선순위 | 버튼 유형 | 예시 | 스타일 |
|---------|----------|------|--------|
| 1 (가장 오른쪽) | Primary CTA | `<i class="bx bx-plus"></i> 등록`, `<i class="bx bx-plus"></i> 신규` | `.btn-primary` |
| 2 | Secondary | `일괄 삭제`, `승인 요청` | `.btn-secondary` or `.btn-outline-*` |
| 3 | 보조 액션 | `가져오기`, `복사` | `.btn-soft-*` |

---

## ZONE-3: 필터 Area

```
┌─────────────────────────────────────────────────────────────────────────┐
│ [A1: 날짜필터] [추가필터-1 Select] [추가필터-2 Select] [검색어 Input]   │
└─────────────────────────────────────────────────────────────────────────┘
```

### A1: 날짜 필터 (공통 필수 컴포넌트)

**기본 형태**: `[전체 📅▾]` — 달력 아이콘 + "전체" 텍스트 + 드롭다운 화살표

**클릭 시 → A1.1 기간조회 팝업 노출**:

```
┌────────────────────────────────┐
│  기간조회                      │
│                                │
│  조회 기간  [최근 6개월      ▾]│
│                                │
│  [YYYY-MM-DD]  ~  [YYYY-MM-DD] │
│          [취소]  [저장]        │
└────────────────────────────────┘
```

| 옵션 | 자동 설정값 | 날짜 입력 활성화 |
|------|-----------|:-------------:|
| 전체 | 제한 없음 | ❌ |
| 최근 6개월 | 오늘 기준 -6개월 ~ 오늘 | ❌ |
| 1년 | 오늘 기준 -1년 ~ 오늘 | ❌ |
| 2년 | 오늘 기준 -2년 ~ 오늘 | ❌ |
| 3년 | 오늘 기준 -3년 ~ 오늘 | ❌ |
| 직접 선택 | — | ✅ From~To 입력 가능 |

> - 날짜 범위 선택 후 **저장** 클릭 시 필터 적용, 날짜 버튼 라벨이 선택 기간으로 변경
> - 기간 선택 상태는 `[2025-09-01 ~ 2025-10-01 📅]` 형태로 표시

### 추가 필터 (Select 드롭다운)

| 요소 | 기본값 | 설명 |
|------|--------|------|
| 추가 필터 Select | `[필터명]: 전체` | 화면별 필요 항목 추가 (예: 생산 공장, 상태, 카테고리) |
| 최대 개수 | 3개 권장 | 초과 시 "더 보기" 필터 패널로 확장 |

### 검색어 Input

| 항목 | 규격 |
|------|------|
| Placeholder | `[검색 가능 컬럼명1] 또는 [검색 가능 컬럼명2]으로 검색하세요.` |
| 검색 트리거 | Enter 키 또는 검색 아이콘 클릭 |
| 검색 대상 | Select 필터를 **제외한** 나머지 텍스트 컬럼 |
| debounce | 300ms (실시간 검색 시) |

---

## ZONE-4: 테이블 상단 도구

```
┌─────────────────────────────────────────────────────────────────────────┐
│  총 N건                                  [테이블 표시 선택: 전체 ▾] [📊] │
└─────────────────────────────────────────────────────────────────────────┘
```

| 요소 | 위치 | 설명 |
|------|------|------|
| `총 N건` | 좌측 | 현재 필터 조건 기준 전체 결과 수 |
| 테이블 표시 선택 | 우측 | 컬럼 가시성 제어 드롭다운 (전체 / 선택 컬럼만) |
| Excel 다운로드 | 우측 끝 | 현재 필터 결과 전체를 Excel로 내보내기 |

---

## ZONE-5: 데이터 테이블

### 컬럼 설계 원칙

| 원칙 | 내용 |
|------|------|
| 필수 컬럼 | ID/코드 (좌측 고정), 이름/명칭, 상태, 수정자, 최근수정일 |
| 링크 컬럼 | 상세 화면으로 이동하는 컬럼은 파란 텍스트 링크 처리 |
| 상태 컬럼 | `사용` / `미사용` 등 — Badge 또는 텍스트 (Soft 컬러 적용) |
| 수정자 | 마지막 수정한 사람 이름 |
| 최근수정일 | `YYYY-MM-DD` 포맷 |
| 정렬 | 컬럼 헤더 클릭으로 오름/내림차순 토글 |
| 행 하이라이트 | 마우스 호버 시 배경색 변경 (`.highlight-on-hover`) |
| 행 클릭 | 상세 화면으로 이동 (링크 컬럼과 동일 동작) |

### 컬럼 너비 가이드

| 컬럼 유형 | 권장 최소 너비 |
|----------|-------------|
| 코드 | 120px |
| 명칭 (링크) | 200px~ (flex 확장) |
| 공장명 (계층 포함) | 160px |
| 상태 | 80px |
| 수정자 | 100px |
| 날짜 | 120px |

### 상태 값 표기 (공통)

| 상태 | 표시 텍스트 | 스타일 |
|------|-----------|--------|
| 활성 / 사용 | `사용` | `.text-success` 또는 기본 텍스트 Bold |
| 비활성 / 미사용 | `미사용` | `.text-muted` |
| 대기 / 검토 | `검토 중` | `.text-warning` |
| 반려 | `반려` | `.text-danger` |

### Empty / 로딩 / 에러 상태

| 상태 | 표시 |
|------|------|
| 데이터 없음 | 중앙에 "조회된 데이터가 없습니다." 텍스트 |
| 로딩 | Spinner + "데이터 불러오는 중..." |
| 에러 | "데이터를 불러오지 못했습니다." + 재시도 버튼 |

## ZONE-6: 페이지네이션

```
┌─────────────────────────────────────────────────────────────────────────┐
│  [50개씩 보기 ▾]                       ←  1  2  3  →                    │
└─────────────────────────────────────────────────────────────────────────┘
```

| 요소 | 규격 |
|------|------|
| 페이지당 표시 수 | **50 / 100 / 200 / 400 (기본: 50)** — 전체 리스트 화면 공통 |
| 페이지 번호 | 현재 페이지 강조 (Primary 색상), 최대 5개 노출 |
| 이전/다음 버튼 | 첫 페이지에서 `←` 비활성, 마지막 페이지에서 `→` 비활성 |

> 📌 **공통 규칙**: 페이지당 표시 수는 `미팅로그 2026-05-12` 결정에 따라 모든 리스트 화면에 동일하게 적용 (50/100/200/400, 기본 50)

---

## 사용 방법 (신규 화면 작성 시)

1. 이 파일을 복사해 `02_기획화면/와이어프레임_[화면명]_목록_v1.md`로 저장
2. `[대괄호]` 항목을 해당 화면에 맞게 교체
3. ZONE-3 추가 필터 Select를 화면에 필요한 항목으로 교체
4. ZONE-5 컬럼 목록을 해당 데이터에 맞게 재정의
5. 화면별 예외 처리가 있는 경우 각 ZONE 하단에 `> 예외:` 블록으로 추가 기재
