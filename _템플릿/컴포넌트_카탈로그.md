# CBAM 기획화면 — 컴포넌트 카탈로그 (조립 가이드)

> 이 파일은 화면을 **조립**할 때 보는 압축 카탈로그다. 거대한 `Layout_Rules.md`(105KB) 대신 이걸 읽는다.
> 스타일은 `_디자인시스템/cbam.css`, 동작 JS는 `_디자인시스템/cbam-proto.js`에 있다(화면에 재방출 금지).
> 화면 HTML은 `_템플릿/기획화면_템플릿.html`(얇은 버전)을 복제해 시작한다.

## 0. 작업 순서
1. `기획화면_템플릿.html` 복제 → `projects/{프로젝트}/02_기획화면/SCR-{ID}_{명}.html`
2. `<title>`·app-logo·topbar(브랜드/메뉴경로/사용자)의 `[PLACEHOLDER]` 교체
3. 하단 inline `<script>`의 **화면별 설정** 수정:
   - `NOTES_STORAGE_KEY='cbam-notes-{ID}'`, `NOTES_EMBED_ID='embeddedNotes{ID}'`, `SAVE_FILENAME`, `<script id="embeddedNotes{ID}">`
   - `SCREEN_IDS=['list','reg',...]` (서브탭 순서와 일치)
   - `INIT_NOTES` (아래 §9 형식)
4. `[SCREEN-CONTENT]`(프로토타입) 채우기 = **1차 산출물**. `[DOC-내용]`(기획문서)은 화면 🟢 확정 시 작성.
5. 검증: 브라우저로 `http://localhost:포트/SCR-...html` 열어 콘솔 0 에러 + 탭/정렬/노트/모달 동작 확인.

> ⚠️ **공유 CSS/JS·노트 프레임워크는 재생성하지 않는다.** 링크만 한다. 인라인 style은 최소화하고 아래 클래스를 쓴다.

---

## 1. 레이아웃 chrome (얇은 템플릿 skeleton에 이미 있음 — 값만 교체)
- `app-tab-bar`: 상단 [📋 기획 문서]/[🖥️ 프로토타입] 전환 (`switchView('doc'|'proto')`)
- `topbar`: 브랜드(`topbar-brand-name/sub`)·메뉴경로·`topbar-user`(avatar/name/role)
- `proto-menu-tree`: GNB 좌측 트리. 활성 항목에 `active`, 펼침에 `open` (`toggleD1/toggleD2`)
- `screen-tab-bar`: 서브화면 탭. `<button class="screen-tab" onclick="switchTab('list',this)">① 목록</button>` (SCREEN_IDS와 일치)
- `lnb-desc-panel#notePanel`: Description 노트 패널 (`toggleNotes()`) — INIT_NOTES가 자동 렌더

## 5. 데이터 그리드 (data-grid) — Type A 정렬 필수
> 모든 `<th>`에 `class="sortable" onclick="sortGrid(컬럼idx)"` + `<span class="sort-icon none"></span>`. 정렬은 `sortGrid`가 처리(공유 JS).
```html

  <table class="data-grid type-a" id="mainGrid">
    <thead><tr>
      <th class="sortable" onclick="sortGrid(0)">제품코드<span class="sort-icon none"></span></th>
      <th class="sortable" onclick="sortGrid(1)">제품명<span class="sort-icon none"></span></th>
      <!-- ... -->
    </tr></thead>
    <tbody>
      <tr>
        <td>MU456456</td>
        <td><span class="td-link" onclick="switchTab('detail',null)">메탈 스크롤 휠</span></td>
        <td style="text-align:center;"><span class="badge b-success">사용</span></td>
      </tr>
    </tbody>
  </table>
</div>
```

## 8. 버튼 · 뱃지 · 모달
**버튼**: `btn` + `btn-primary` / `btn-outline` / `btn-outline-primary` / `btn-outline-danger` / `btn-soft` / `btn-soft-primary` / `btn-dark` / `btn-icon`. 크기 `btn-sm`.
**뱃지**: `badge` + `b-success`/`b-warning`/`b-danger`/`b-info`/`b-primary`. 문서 상태는 `badge-doc`, 사각형은 `badge-rect`.
**모달**: 표시 `showModal('id')` / 닫기 `closeModal('id')`.
```html
<div class="modal-overlay" id="saveModal">
  <div class="modal-container">
    <div class="modal-header"><span class="modal-title">저장하시겠습니까?</span>
      <button class="modal-close" onclick="closeModal('saveModal')">✕</button></div>
    <div class="modal-body">입력한 내용을 저장합니다.</div>
    <div class="modal-footer">
      <button class="btn btn-outline" onclick="closeModal('saveModal')">취소</button>
      <button class="btn btn-primary" onclick="closeModal('saveModal')">확인</button>
    </div>
  </div>
</div>
<!-- 넓은 조회 모달: <div class="modal-container" style="width:min(960px,90vw);"> + modal-body 안에 search-input-wrapper + data-grid -->
```

## 9. Description 노트 (INIT_NOTES) — body는 `<ul><li>` 필수
```js
const INIT_NOTES = {
  list:[
    {n:1, name:'검색', body:'<ul><li>안내문구 : 제품코드/제품명으로 검색하세요</li><li>기본값 : null</li></ul>'},
    {n:2, name:'데이터 그리드', body:'<ul><li>기본정렬 : 최근수정일 내림차순</li><li>정렬 : 모든 컬럼</li></ul>'},
  ],
  reg:[ {n:1, name:'CN 코드', body:'<ul><li>필수</li><li>[CN 코드 조회] 팝업</li></ul>'} ],
};
```
- 화면 요소에 **배지 앵커**를 붙이려면 해당 요소에 `data-widget="list1"`처럼 (탭+번호) 고유 id를 주고, INIT_NOTES 항목과 번호를 맞춘다. (배지는 노트 패널 열 때 `renderNoteBadges`가 표시)
