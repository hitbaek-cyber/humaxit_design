# 프로젝트별 디자인 토큰 사용 가이드

> 이 문서는 `base.css`(공통 컴포넌트)와 `tokens/*.css`(프로젝트별 색상)를 분리 운영하는 방법을 설명합니다.

---

## 구조

```
_디자인시스템/
├── base.css              ← 공통 컴포넌트 스타일 (버튼·모달·그리드·폼 등). 색상은 변수로만 참조.
├── cbam.css               ← (호환용 래퍼) tokens/cbam.css + base.css를 import. 기존 CBAM HTML 호환 유지.
├── tokens/
│   ├── cbam.css           ← CBAM_SaaS 프로젝트 색상 토큰
│   └── law_ai.css         ← Law_ai_DL 프로젝트 색상 토큰
├── Design_Tokens.md        ← 컴포넌트 스펙 문서 (치수·구조 — 프로젝트 무관)
├── Layout_Rules.md
└── ...
```

## 원칙

- **base.css**는 프로젝트와 무관한 컴포넌트 스타일만 담습니다 (버튼 높이 38px, 모달 z-index 등). 색상은 전부 `var(--color-primary)` 같은 CSS 변수로만 참조하고, 실제 색상값(`:root`)은 선언하지 않습니다.
- **tokens/{project}.css**는 `:root` 안에 해당 프로젝트의 색상·폰트 변수만 선언합니다. 컴포넌트 스타일은 작성하지 않습니다.
- 신규 프로젝트를 시작할 때는 `tokens/{project명}.css`를 새로 만들고, 기존 프로젝트(CBAM, Law_ai)의 토큰 파일을 참고해 변수명을 동일하게 맞춥니다.

## 새 HTML에서 사용하는 법

```html
<head>
  <link rel="stylesheet" href="../../_디자인시스템/tokens/law_ai.css">
  <link rel="stylesheet" href="../../_디자인시스템/base.css">
</head>
```

- `tokens/*.css`를 **반드시 base.css보다 먼저** 로드해야 합니다 (CSS 변수 선언 → 사용 순서).
- 기존 CBAM_SaaS 산출물처럼 `cbam.css` 한 파일만 import하던 방식도 계속 동작합니다 (호환용 래퍼가 내부적으로 두 파일을 불러옵니다).

## _템플릿 폴더 사용하는 법

`_템플릿/` 안의 5개 템플릿(PRD, 화면흐름도 뷰어, 히스토리 뷰어, 기획화면 통합, 기획화면 개별)은 모두 색상을 `:root` 직접 선언 대신 `[[TOKENS_CSS_PATH]]` 플레이스홀더로 받습니다.

1. 템플릿을 `projects/{프로젝트}/{단계폴더}/`로 복사
2. `[[TOKENS_CSS_PATH]]`를 해당 프로젝트의 토큰 파일 상대경로로 치환
   - 예: `projects/Law_ai_DL/01_PRD/PRD_v1_전체개요.html`에서는 `../../_디자인시스템/tokens/law_ai.css`
3. 그 외 `[[버전]]`, `[[개요명]]`, `[SCREEN-ID]` 등 나머지 플레이스홀더를 실제 내용으로 교체

각 템플릿이 사용하는 CSS 변수 체계가 조금씩 다르므로(`--color-*`, `--primary` 등, `--humax-*`), **tokens/*.css 파일에는 3가지 체계의 별칭이 모두 선언되어 있습니다.** 새 프로젝트 토큰을 추가할 때는 `tokens/cbam.css` 또는 `tokens/law_ai.css`를 참고해 3개 섹션을 모두 채워주세요.

| 템플릿 | 사용 변수 체계 |
|--------|---------------|
| 기획화면_개별_템플릿.html | `--color-*` (base.css와 동일) |
| 기획화면_통합_템플릿.html | `--color-*` |
| PRD_템플릿.html | `--primary`, `--dark`, `--text` 등 |
| 화면흐름도_뷰어_템플릿.html | `--primary`, `--dark`, `--text` 등 |
| 히스토리_뷰어_템플릿.html | `--humax-*` |

## 프로젝트별 토큰 값 비교

| 변수 | CBAM_SaaS | Law_ai_DL |
|------|-----------|-----------|
| `--color-primary` | `#0561A4` | `#2151EC` |
| `--color-success` | `#006B56` | `#1BC47D` |
| `--color-danger` | `#FF5812` | `#EA5455` |
| `--color-warning` | `#CEA331` | `#F0AF23` |
| `--font-family-base` | `'NotoSansKR'` | `'Pretendard'` |

## 컴포넌트 스펙 문서(Design_Tokens.md)와의 관계

`Design_Tokens.md`는 컴포넌트의 **구조·치수·동작**을 설명하는 문서로, 프로젝트와 무관하게 공통 적용됩니다 (예: "버튼 높이 38px", "모달 padding 28px"). 색상은 이 문서에서도 `var(--color-primary)` 같은 변수명으로만 언급되며, 실제 값은 `tokens/*.css`를 따릅니다.

CBAM 전용 컴포넌트(§13 Hierarchical Tree, §16 Formula Detail 등 — 탄소배출 계산식·계층구조)는 `base.css`에 포함되어 있지 않습니다. 이런 프로젝트 전용 컴포넌트는 각 프로젝트 폴더 안에 별도 문서로 관리하는 것을 권장합니다 (예: `projects/CBAM_SaaS/_design-ext.md`).
