## 1. 개요

| 항목 | 내용 |
|------|------|
| 대상 기능 | 법률자문 상세 조회 — 코멘트 추가 / AI 코멘트 자동 생성 |
| 진입점 | 법률자문 요청 등록 또는 법률자문 조회 목록에서 건 클릭 |
| 종료점 | 코멘트 저장 완료 / 상세 조회 화면 닫기 |
| 관련 사용자 | 자문 요청자, 법무 담당자 |

> ⚠️ 본 흐름도는 지금까지 확인된 요구사항(코멘트 추가 UX 개선, AI 코멘트 자동 생성)을 기반으로 추정 작성되었습니다. 실제 화면 동작과 차이가 있을 수 있어 검증이 필요합니다.

## 2. 전체 흐름

```mermaid
flowchart TD
    A([법률자문 메뉴 진입]) --> B{진입 경로}
    B -->|요청 등록 후 저장| C[법률자문 상세 조회]
    B -->|조회 목록에서 건 클릭| C

    C --> D[자문 요청 내용 + 코멘트 영역\n동시 노출]
    D --> E{코멘트 추가 클릭}
    E --> F[코멘트 작성 패널 오픈\n자문 내용 가림 없이 동시 열람]
    F --> G{AI 코멘트 생성 버튼 클릭?}
    G -->|예| H[AI가 요청 제목/본문/첨부/\n이전 코멘트 이력/수신자 정보 기반 초안 생성]
    H --> I[리치텍스트 에디터에\n생성된 초안 자동 입력]
    I --> J[사용자가 초안 검토·수정]
    G -->|아니오| J
    J --> K{저장}
    K -->|저장| L[코멘트 등록 완료\n상세 화면 갱신]
    K -->|취소| D
    L --> M([종료 또는 다른 자문 건 이동])
```

## 3. 화면별 상세

---

### SCR-LAW-001 — 법률자문 AI코멘트

| 항목 | 내용 |
|------|------|
| 진입 조건 | GNB > 법률자문 > 법률자문 조회 > 상세 |
| 화면 목적 | 법률자문 상세 내용을 열람하며 동시에 코멘트를 작성, AI를 통한 코멘트 자동 생성 |

```mermaid
flowchart TD
    S1([SCR-LAW-001 진입]) --> S2[자문 요청 내용 영역 표시\n제목/본문/첨부/수신자]
    S2 --> S3{코멘트 추가 클릭}
    S3 --> S4[코멘트 작성 패널 오픈\n자문 내용 영역과 동시 노출]
    S4 --> S5{AI 코멘트 생성 클릭?}
    S5 -->|예| S6[AI 초안 생성 요청]
    S6 --> S7{생성 결과}
    S7 -->|성공| S8[에디터에 초안 자동 입력]
    S7 -->|실패/오류| S9[오류 안내 + 수동 작성 유도]
    S8 --> S10[사용자 검토·수정]
    S9 --> S10
    S5 -->|아니오| S10
    S10 --> S11{저장 확인}
    S11 -->|저장| S12[코멘트 등록 완료\n목록/이력 갱신]
    S11 -->|취소| S4
    S12 --> S13([종료 또는 다른 메뉴 이동])
```

**가능한 액션**

| 사용자 액션 | 다음 화면/상태 | 조건 |
|------------|-------------|------|
| 코멘트 추가 클릭 | 코멘트 작성 패널 오픈 (자문 내용 가리지 않음) | — |
| AI 코멘트 생성 클릭 | AI 초안 생성 → 에디터 자동 입력 | 자문 요청 내용 존재 시 |
| 초안 수정 | 에디터 내용 갱신 | — |
| 저장 | 코멘트 등록 완료 | 필수 입력값 충족 시 |
| 취소 | 코멘트 작성 패널 닫힘 | — |

**에러/예외**

| 케이스 | 처리 |
|--------|------|
| AI 생성 실패 | 오류 안내 메시지 + 수동 작성으로 전환 |
| 필수값 미입력 후 저장 시도 | 인라인 오류 표시 |
