### T-005 · 최초 진입 시 인덱스+프로토타입 동시 노출 버그 수정

- **상태**: 🟢 완료
- **요청일**: 2026-06-30 | **완료일**: 2026-06-30
- **요청자**: 백교민

**요청 내용**
HTML 최초 오픈 시 인덱스 탭 화면 우측에 프로토타입(법률자문 조회 페이지)이 겹쳐서 노출되고 사이드바도 눌려 보임. 다른 탭으로 이동했다가 재진입하면 정상적으로 인덱스만 보임.

**원인 분석**
- `<div class="view proto-view active" id="view-proto">`에 초기 HTML 작성 시 실수로 `active` 클래스가 남아있었음
- `<div class="view index-view active" id="view-index">`도 `active`였기 때문에, 최초 로드 시 `.view.active{display:flex}` 규칙에 의해 두 뷰가 동시에 `display:flex`로 렌더링됨
- `switchView()`로 다른 탭을 한 번이라도 클릭하면 `.view`에서 active를 전부 제거한 뒤 선택된 탭에만 다시 붙이므로, 그 이후로는 정상적으로 1개 뷰만 보였던 것

**작업 계획**
- [x] `view-index`/`view-doc`/`view-proto` 각각의 초기 class 속성 전수 확인
- [x] `view-proto`의 불필요한 `active` 클래스 제거
- [x] jsdom으로 페이지 로드 직후(추가 클릭 없이) 3개 view의 active 상태 검증 — `view-index`만 true, 나머지 false, JS 에러 0건 확인

**결과/변경된 파일**
- 수정: [SCR-LAW-001_법률자문_ai코멘트.html](../02_기획화면/SCR-LAW-001_법률자문_ai코멘트.html)
- 수정: [SCR-LAW-001_법률자문_ai코멘트.md](../02_기획화면/SCR-LAW-001_법률자문_ai코멘트.md) (변경이력 v1.4 추가)

**비고**: T-002에서 탭바 구조를 이식할 때 발생한 초기 마크업 실수였음. 향후 새 화면에 동일 패턴을 적용할 때는 `.view.active`가 정확히 1개만 존재하는지(`grep -c 'view.*active'` 등으로) 사전 점검하는 것을 권장.

### T-004 · Description 버튼 무반응 버그 수정 + 노출 범위 변경(app-shell 전체)

- **상태**: 🟢 완료
- **요청일**: 2026-06-30 | **완료일**: 2026-06-30
- **요청자**: 백교민

**요청 내용**
1. Description 버튼 클릭해도 패널이 열리지 않는 버그 수정
2. 노출 방법 변경 — AS IS: `content-area-row`(topbar 제외) 기준 사이드시트 → TO BE: `app-shell`(topbar 포함) 기준 사이드시트

**원인 분석**
- 1차 원인(이전 작업에서 잔존): `toggleNotes()` 함수가 두 번 정의되어 있었고, 구버전(`descPanel`이라는 옛 ID 참조)이 나중에 로드되어 신버전을 덮어씀 → 별도 커밋에서 제거 완료된 상태였음
- 2차 원인(이번 발견): `const _descOrigShowPage = showPage;` 코드가 Description 스크립트 블록 안에서 **즉시 실행**되는데, 원본 `function showPage(){}` 정의는 그 다음 `<script>` 블록에 있어 아직 등록되지 않은 시점이었음. JS의 `function` 호이스팅은 같은 `<script>` 블록 내에서만 보장되며 다른 블록까지 끌어올려지지 않음 → `showPage`가 `undefined`로 참조되어 이후 `toggleNotes` 호출 체인에서 연쇄 에러 발생

**작업 계획**
- [x] jsdom으로 실제 클릭 시뮬레이션 실행, 콘솔 에러 재현 (`Cannot access 'descStore' before initialization` 등)
- [x] `_descOrigShowPage = showPage` 즉시 실행 코드를 `DOMContentLoaded` 리스너 안으로 이동 (모든 script 블록 로드 후 안전하게 wrap)
- [x] 중복 wrap 방지 플래그(`showPage.__descWrapped`) 추가
- [x] Description 패널을 `content-area-row`(topbar 아래) 밖으로 빼서 `app-shell`의 직계 자식으로 이동
- [x] `app-shell`을 `flex-direction: row`로, `topbar`+`content-area`를 묶는 `app-main-col`(column) wrapper 신설
- [x] jsdom 재검증: 토글 열기/닫기 2회, 페이지 전환 시 노트 갱신까지 에러 0건 확인

**결과/변경된 파일**
- 수정: [SCR-LAW-001_법률자문_ai코멘트.html](../02_기획화면/SCR-LAW-001_법률자문_ai코멘트.html)
- 수정: [SCR-LAW-001_법률자문_ai코멘트.md](../02_기획화면/SCR-LAW-001_법률자문_ai코멘트.md) (변경이력 v1.3 추가)

**비고**: 이번엔 시각적 확인뿐 아니라 jsdom + node로 실제 클릭 이벤트를 시뮬레이션하여 JS 에러 0건을 직접 검증함. 노출 범위 변경으로 패널이 펼쳐질 때 topbar(검색바·사용자 정보)는 그대로 유지되고, 좌측 sidebar+topbar+content-area 전체 폭이 줄어들며 우측에 패널이 펼쳐지는 구조가 됨.
