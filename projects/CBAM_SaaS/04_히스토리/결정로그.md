### D-003 | CBAM 인증서 구매 수량 계산 기능 방향

| 항목 | 내용 |
|------|------|
| 결정일 | 2026-04-28 |
| 결정자 | 대표님 |
| 참여자 | 왕기창팀장, 노준호팀장, 전정화책임, 임지현책임 |
| 관련 기능 | CBAM 모듈 — 내재배출량 기반 인증서 구매 수량 계산 |
| 관련 미팅 | [2026-04-28 미팅노트](미팅노트/2026-04-28_CBAM모듈기능추가_기지불탄소가격_AI서비스툴.md) |

**배경 / 문제 상황**
> 고객 관점에서 CBAM 인증서 구매 비용을 예측하는 기능이 최우선 개발 항목으로 요구됨. 내재배출량 산정 결과를 CBAM 인증서 구매 수량으로 연결하는 계산 흐름 정의가 필요.

**최종 결정**
> ✅ 아래 4단계 흐름으로 CBAM 인증서 구매 수량 계산 기능 구현:
> 1. 제품의 고유 **내재 배출량** 산정
> 2. 데이터의 실제/기본값에 따른 **Benchmark 기준값** 적용
> 3. **기지불탄소가격은 우선 0으로 가정** (법안 확정 전)
> 4. **분기별 CBAM 인증서 가격** 기준으로 구매 수량 계산

**결정 근거**
- 내재배출량 → 비용 가시성 제공이 영업 관점 최우선 고객 가치
- 기지불탄소가격 이행 법안 미확정 상태이므로 0 가정으로 단순화
- 분기별 CBAM 인증서 가격은 수동 등록으로 관리 가능 (→ D-001 연계)

### D-002 | 기지불탄소가격 적용 방식 — 우선 0 가정, 법안 확정 후 Deep Dive

| 항목 | 내용 |
|------|------|
| 결정일 | 2026-04-28 |
| 결정자 | 노준호팀장 |
| 참여자 | 대표님, 왕기창팀장, 전정화책임, 임지현책임 |
| 관련 기능 | CBAM 모듈 — 기지불탄소가격 산정 |
| 관련 미팅 | [2026-04-28 미팅노트](미팅노트/2026-04-28_CBAM모듈기능추가_기지불탄소가격_AI서비스툴.md) |

**배경 / 문제 상황**
> '26년 5월 제 3국 기지불탄소가격에 대한 이행 법안 Release 예정이나, 법안 최종 확정은 2026년 하반기로 예상됨. 법안 미확정 상태에서 기지불탄소가격 계산 로직을 설계하면 추후 대규모 수정이 필요할 수 있음.

**검토한 옵션들**

| 옵션 | 장점 | 단점 |
|------|------|------|
| 법안 초안 기준으로 즉시 구현 | 빠른 기능 제공 | 법안 변경 시 재개발 비용 발생 |
| 기지불탄소가격 0 가정 후 법안 확정 시 추가 구현 | 안정적, 불필요한 재개발 방지 | 당장 정확한 수치 미반영 |

**최종 결정**
> ✅ **기지불탄소가격은 우선 0으로 가정**하여 계산. 이행 법안 최종 확정(2026년 하반기 예정) 시 Deep Dive 후 기능 추가 구현.

**결정 근거**
- 현재 법안 초안 기준으로는 국내 무상 할당량 인정 여부 등 세부 사항 불명확
- 법안 확정 전 구현 시 재개발 리스크 과도
- 기지불탄소가격 = 0 가정해도 고객에게 내재배출량 기반 인증서 구매 수량 가시성 제공 가능

### D-001 | CBAM 벤치마크·인증서 가격 데이터 관리 방식 (엑셀 등록)

| 항목 | 내용 |
|------|------|
| 결정일 | 2026-04-28 |
| 결정자 | |
| 참여자 | 대표님, 왕기창팀장, 노준호팀장, 전정화책임, 임지현책임 |
| 관련 기능 | CBAM 모듈 — CN코드별 벤치마크 조회, CBAM 인증서 가격 관리 |
| 관련 미팅 | [2026-04-28 미팅노트](미팅노트/2026-04-28_CBAM모듈기능추가_기지불탄소가격_AI서비스툴.md) |

**배경 / 문제 상황**
> CBAM 모듈에서 CN코드별 벤치마크 값 및 CBAM 인증서 가격을 조회하는 기능이 필요. EU 집행위원회 API 연동 방식과 수동 등록 방식 중 결정이 필요함.

**검토한 옵션들**

| 옵션 | 장점 | 단점 |
|------|------|------|
| EU 집행위원회 API 연동 | 자동화, 실시간 반영 | 현재 API 없음, 개발 복잡도 높음 |
| 엑셀 등록 수동 관리 | 빠른 구현, 유연한 관리 | 주기적 수동 업데이트 필요 |

**최종 결정**
> ✅ **엑셀 등록 수동 관리 방식 채택**
> - CN코드별 벤치마크 값: 자주 변경되지 않으므로 엑셀 등록으로 충분
> - CBAM 인증서 가격: 현재 분기별 특정 사이트에서 제공, 엑셀 등록으로 관리
> - EU API 인터페이스는 당장 고려하지 않음 ('27년부터 CBAM Registry API 가능성 검토 예정)

**결정 근거**
- EU 집행위원회에서 현재 제공하는 API 없음
- CN코드별 벤치마크 값은 자주 변경되지 않아 수동 관리 가능
- CBAM 인증서 가격은 분기별 제공으로 수동 등록 부담 낮음
- '27년부터 CBAM Registry API 제공 가능성 있어 추후 자동화 여지 있음
