# 기획 Tool 프로젝트 관리 허브

> **목적**: 프로젝트별 기획서, 기획화면, 화면흐름도, 의사결정 히스토리를 체계적으로 관리합니다.

---

## 📁 폴더 구조

```
기획Tool프로젝트/
├── README.md                       ← 지금 보고 있는 파일 (전체 인덱스)
│
├── _디자인시스템/                  ← 전 프로젝트 공통 컴포넌트/토큰
│   ├── 컴포넌트/                   ← 버튼, 인풋, 카드 등 UI 컴포넌트 정의
│   ├── 토큰/                       ← 색상, 타이포그래피, 간격 등 디자인 토큰
│   └── 가이드라인/                 ← 사용 규칙 및 원칙
│
├── _템플릿/                        ← 새 문서 작성 시 복사해서 사용
│   ├── PRD_템플릿.md
│   ├── PRD_템플릿.html               ← PRD 사이드바 목차형 뷰어
│   ├── 화면흐름도_템플릿.md
│   ├── 화면흐름도_뷰어_템플릿.html   ← Mermaid 데이터 체크 내장
│   ├── 결정사항_템플릿.md
│   ├── 미팅노트_템플릿.md
│   └── 히스토리_뷰어_템플릿.html
│
└── projects/                       ← 프로젝트별 폴더 (폴더명 영문 권장, 예: CBAM_SaaS)
    └── [프로젝트명]/
        ├── 01_PRD/                ← PRD (.md + .html 한 쌍)
        ├── 02_기획화면/            ← 화면 기획 문서 (SCR-*.html)
        ├── 03_화면흐름도/          ← 화면흐름도 (.md + .html 한 쌍)
        └── 04_히스토리/            ← 결정로그, 히스토리 뷰어, 미팅노트
            └── 미팅노트/
```

> 프로젝트 단위 `README.md`는 더 이상 필수가 아니며, `01_PRD` 의 PRD 문서가 개요 역할을 한다.

## 📌 문서 작성 규칙

### 파일 네이밍
- PRD: `PRD_v{버전}_{개요명}.md` + 동명 `.html` (예: `PRD_v1_전체개요.md` / `.html`)
- 화면흐름도: `화면흐름_{기능명}_v{버전}.md` + 동명 `.html`
- 기획화면: `SCR-{화면ID}_{화면명}.html` (예: `SCR-CBAM-001_벤치마크.html`)
- 미팅노트: `{YYYY-MM-DD}_{회의주제}.md`
- 결정사항: `결정로그.md` (누적 기록)

> 화면 ID·기능 ID에는 프로젝트 코드를 넣는다 (예: `SCR-CBAM-001`, `F-CBAM-01`). 자세한 규칙은 `CLAUDE.md` 참조.

### 버전 관리
- 파일명에 `_v{숫자}` 로 버전 표기
- 대규모 변경 시 새 파일 생성, 소규모 수정 시 동일 파일 내 변경 이력 추가

### 상태 표기
문서 상단에 아래 뱃지 중 하나를 사용하세요:
- `🟡 초안` — 작성 중
- `🔵 검토 중` — 피드백 수렴 중
- `🟢 확정` — 최종 승인 완료
- `🔴 폐기` — 더 이상 유효하지 않음
